# Web Dashboard 设计规格

## 目标

为币安合约分析系统构建一个 Web 前端页面，兼顾实时监控和手动操作两种使用场景。

## 技术方案

- 纯静态 HTML + Vanilla JS + CSS，无构建工具
- 文件放在 `src/main/resources/static/` 下，Spring Boot 自动托管
- 亮色主题

## 文件结构

```
src/main/resources/static/
├── index.html
├── css/
│   └── style.css
└── js/
    └── app.js
```

## 页面布局

### 顶部栏
- 系统标题："币安合约分析系统"
- 健康状态指示灯（调用 `/api/health`）
- 合约总数显示（调用 `/api/analysis/symbols`）

### 左侧栏（固定宽度 220px）
- "总览" 入口（默认选中）
- 分析器列表，每项显示：
  - 分析器名称
  - 最新结果数量 badge
- 底部：刷新合约列表按钮

### 右侧内容区

#### 总览页（默认）
- 网格卡片展示所有分析器的最新结果摘要
- 每张卡片：分析器名称、结果数量、更新时间、前3个币种
- 点击卡片跳转到对应分析器详情

#### 分析器详情页
- **参数区域**：根据 `/api/analysis/params/{name}` 返回的参数说明动态渲染表单
  - 每个参数显示：名称、类型、默认值、输入框
  - "执行分析" 按钮（调用 `POST /api/analysis/sync/{name}`）
  - "重置为默认" 按钮
- **结果区域**：表格展示分析结果
  - 列：币种、涨幅/评分、关键指标（根据分析器类型不同）
  - 支持按涨幅/评分排序
  - 空结果时显示"无符合条件的币种"
- **状态提示**：执行中显示 loading，失败显示错误信息

## 数据刷新策略

| 分析器 | 刷新方式 | 频率 |
|--------|----------|------|
| 短期急涨 | 自动轮询 | 每 30 秒 |
| 其他所有 | 手动触发 | 用户点击执行 |

- 页面加载时自动拉取一次所有最新结果（`/api/analysis/latest`）
- 左侧栏 badge 跟随数据更新
- 短期急涨轮询可通过页面上的开关暂停/恢复

## API 调用映射

| 页面操作 | API 端点 | 方法 |
|----------|----------|------|
| 页面加载 | `/api/health` | GET |
| 页面加载 | `/api/analysis/list` | GET |
| 页面加载 | `/api/analysis/latest` | GET |
| 点击分析器 | `/api/analysis/params/{name}` | GET |
| 点击分析器 | `/api/analysis/latest/{name}` | GET |
| 执行分析 | `/api/analysis/sync/{name}` | POST |
| 刷新合约 | `/api/analysis/symbols/refresh` | POST |
| 获取合约数 | `/api/analysis/symbols` | GET |

## 交互流程

1. 页面加载 → 并行调用 health、list、latest 接口
2. 渲染左侧栏分析器列表 + 总览页卡片
3. 用户点击左侧分析器 → 加载参数说明和最新结果，渲染详情页
4. 用户修改参数并点击"执行" → POST 请求带参数，展示返回结果
5. 短期急涨自动轮询 → 每30秒更新结果和左侧 badge

## 错误处理

- API 不可达：顶部状态灯变红，显示"服务离线"
- 请求超时（10秒）：显示超时提示，允许重试
- 频率限制（后端10秒限制）：按钮禁用倒计时
- 参数校验：前端基础校验（数字类型、非空）

## 样式规范

- 字体：系统默认字体栈
- 主色调：蓝色系（#1890ff）
- 背景：白色 (#fff) / 浅灰 (#f5f5f5)
- 圆角：4px
- 响应式：最小宽度 1024px，不考虑移动端
